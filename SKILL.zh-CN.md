# Negative-First Mesh Reasoning

[한국어](SKILL.ko.md) | [English](SKILL.en.md) | [日本語](SKILL.ja.md) | **[简体中文](SKILL.zh-CN.md)**

## 1. 目的

本技能不会在收到问题后立即生成最可能的答案，而是按照以下顺序运行。

1. 将问题拆解为可验证条件与语义轴。
2. 将可能的答案、解释、原因、文档和目标激活为候选节点。
3. 对全部候选项应用否定门，优先关闭明显不成立的候选项。
4. 如果没有候选项存活，则把搜索扩展到下一个相关候选组。
5. 只有在足够证据存活时才生成答案。
6. 如果经过合理扩展仍无证据，不得编造内容，应明确表示不存在或不确定。

核心原则：

> 先排除不成立的内容，再陈述剩余内容。

---

## 2. 使用条件

以下任务应优先使用本技能。

- 核验某项主张是否属实
- 存在多种可能解释的问题
- 根因分析与故障诊断
- 搜索文件、邮件、文档与数据
- 比较技术或产品
- 查找满足指定条件的对象
- 法律、医疗、金融等错误结论风险较高的问题
- 需要最新信息的问题
- AI可能在没有证据时自行填补空白的问题
- 需要准确区分“不存在”“不知道”和“不可能”的问题

以下任务不应过度使用本技能。

- 纯创作
- 对用户提供文本进行简单翻译或校对
- 答案明确的简单计算
- 仅要求自由发散创意的任务

---

## 3. 状态模型

每个候选节点必须且只能具有以下状态之一。

| 状态 | 含义 |
|---|---|
| `ACTIVE_SUPPORTED` | 有证据支持，保留为回答候选项 |
| `ACTIVE_TENTATIVE` | 有部分证据，但仍需进一步验证 |
| `OFF_FALSE` | 明确不符合问题条件 |
| `OFF_CONTRADICTED` | 被更强证据或内部矛盾排除 |
| `OFF_OUT_OF_SCOPE` | 因超出当前验证范围而排除 |
| `OFF_UNSUPPORTED` | 因缺乏足够证据而排除 |
| `UNKNOWN_UNRESOLVED` | 现有资料不足以判断真伪 |
| `NOT_FOUND_IN_SCOPE` | 已搜索定义明确的闭合范围，但未发现 |

不得把 `OFF_UNSUPPORTED` 转换为 `OFF_FALSE`。  
证据不足并不等于已经证明错误。

---

## 4. 执行流程

### 步骤A. 问题规范化

将问题拆解为以下要素。

- `target`: 要查找或判定的对象
- `claim`: 要验证的主张
- `constraints`: 必须满足的条件
- `exclusions`: 必须排除的条件
- `scope`: 数据、时间、地区、文档和系统范围
- `freshness`: 是否需要最新验证
- `risk`: 错误回答可能造成的风险
- `answer_type`: 事实、估计、推荐、设计或诊断

不得把搜索范围无限扩展到用户未提出的领域。  
如果缺少必要范围但无需追问也能继续，应声明合理的默认范围后执行。

### 步骤B. 生成网状轴

不要把候选项仅作为一列清单，而应沿不同轴展开。

默认轴：

1. 精确匹配
2. 同义词与相似表达
3. 上位与下位分类
4. 时间差异
5. 地区与对象差异
6. 原因与结果
7. 反例与矛盾
8. 来源与可信度
9. 用户问题中的隐含前提
10. 替代解释

在条件允许时，应批量搜索、并行检索并同时比较这些轴上的候选项。

### 步骤C. 优先执行否定门

在生成答案前，对全部候选项优先应用以下门控。

#### 1）定义门
候选项与问题中的目标是否具有相同定义？

如果不同，设为 `OFF_FALSE` 或 `OFF_OUT_OF_SCOPE`。

#### 2）条件门
候选项是否明确违反任一必需条件？

如果违反，立即设为 `OFF_FALSE`。

#### 3）时间门
证据时间是否符合请求时间？

如果资料过时或属于不同时间段，设为 `OFF_OUT_OF_SCOPE` 或 `ACTIVE_TENTATIVE`。

#### 4）存在门
目标是否实际存在于可验证的来源、文件或数据中？

在闭合范围内不存在时，设为 `NOT_FOUND_IN_SCOPE`。  
对于整个互联网或现实世界等开放范围，只能设为 `UNKNOWN_UNRESOLVED`。

#### 5）矛盾门
候选项是否与已知事实、用户条件或更强证据相矛盾？

如果矛盾，设为 `OFF_CONTRADICTED`。

#### 6）证据门
是否存在足够直接的证据，可将候选项写入回答？

如果没有，设为 `OFF_UNSUPPORTED`。

#### 7）因果门
是否把相关性错误地表述为因果关系？

如果因果关系未被验证，将原因候选项降为 `ACTIVE_TENTATIVE` 或更低。

#### 8）安全门
错误结论是否可能带来重大风险？

如果风险较高，没有更强证据时不得给出确定结论。

### 步骤D. 零检测

检查是否所有候选项都已关闭。

#### 存在一个或多个活跃候选项
重新检查存活候选项的证据强度与相互矛盾，然后生成回答。

#### 活跃候选项为零
不得生成看似合理的填充内容。必须区分：

- 条件自身相互矛盾 → `CONTRADICTED_QUERY`
- 在闭合验证范围内未找到 → `NOT_FOUND_IN_SCOPE`
- 当前搜索范围不足 → 扩展到下一候选组
- 资料不足，无法判断 → `UNKNOWN_UNRESOLVED`
- 主张被直接反证 → `FALSE`

### 步骤E. 扩展到下一候选组

仅当第一个候选组无法得出答案时，按照以下顺序扩展。

1. 精确表达
2. 拼写与记法变体
3. 同义词与缩写
4. 上位与下位概念
5. 邻近时间段
6. 邻近数据源
7. 反向搜索
8. 用结果搜索原因，或用原因搜索结果
9. 对用户前提的替代解释

每次扩展都记录与原问题之间的语义距离。  
不得把距离过远的候选项当作精确答案呈现。

### 步骤F. 生成回答

只能使用剩余的 `ACTIVE_SUPPORTED`，以及在必要时明确标注的
`ACTIVE_TENTATIVE` 来生成回答。

默认顺序：

1. 直接回答
2. 判定状态
3. 已应用的验证范围
4. 核心证据
5. 已排除的主要误解或候选项
6. 剩余不确定性
7. 下一步需要验证的候选组

不得向用户暴露完整内部推理或隐藏思维过程。  
只显示简短、可验证的排除摘要与证据。

## 5. 不存在与未知的判定规则

严格区分以下表述。

### “该主张是错误的”
仅在反例或直接证据证明该主张不成立时使用。

### “在验证范围内未找到”
仅在搜索对象属于明确闭合范围时使用。

示例：

- 用户提供的3份文档
- 特定数据库
- 指定日期范围
- 用户的收件箱
- 特定代码仓库的当前分支

### “无法确认”
在没有资料或无法访问资料，因而不能判断时使用。

### “可能性较低”
在没有直接反证，但候选项与多个条件不匹配时使用。

### “未知”
在合理探索到后续候选组后，仍没有足够判定依据时使用。
不得仅因第一次搜索没有结果就使用。

## 6. 可信度计算

内部可将以下项目按0到1评分。

- `D`: 直接证据
- `S`: 来源可信度
- `C`: 条件满足度
- `R`: 时效性
- `X`: 矛盾惩罚
- `G`: 问题与候选项之间的距离惩罚

建议分数：

`confidence = D × S × C × R × (1 - X) × (1 - G)`

不得声称该分数是绝对概率。

建议判定：

- `0.85以上`: 强证据
- `0.65～0.84`: 很可能
- `0.40～0.64`: 不确定
- `低于0.40`: 从回答候选项中排除，或仅标为推测

高风险领域应使用更高阈值。

## 7. 工具使用规则

### 网页
对于最新信息、法律、价格、日程、政治人物或公职人员、软件版本和当前公司信息，不得依靠记忆，应验证最新来源。

### 文件
当问题依赖特定文件、文档或附件时，应优先搜索原始文件，而不是网页。

### 邮件、日历与代码仓库
对于用户私人数据，只能在相关连接工具实际可访问的范围内确认“未找到”。

### 搜索优化
工具允许时，应批量检索候选组。
即使调用必须顺序执行，在逻辑上也应对同一候选集合应用相同的否定门。

### 引用
应在事实主张附近标注支持它的核心证据。
不得把没有证据的推测表述为事实。

## 8. 用户回答格式

### 默认格式

```text
[回答]
证据最强的结论。

[判定]
已确认 / 很可能 / 不确定 / 验证范围内未找到 / 无法判断 / 错误

[范围]
检查了哪些内容，以及验证到什么程度。

[优先排除项]
一至三个主要的非匹配候选项。

[剩余不确定性]
仍未得到验证的部分。
```

如果用户要求简短回答，只需自然地提供直接答案和会实质影响结果的不确定性。

### 需要JSON输出时

```json
{
  "answer": "",
  "status": "CONFIRMED | LIKELY | UNCERTAIN | NOT_FOUND_IN_SCOPE | UNKNOWN | FALSE",
  "scope": [],
  "active_candidates": [],
  "disabled_candidates": [
    {
      "candidate": "",
      "state": "OFF_FALSE | OFF_CONTRADICTED | OFF_OUT_OF_SCOPE | OFF_UNSUPPORTED",
      "reason": ""
    }
  ],
  "evidence": [],
  "confidence": 0.0,
  "next_group": []
}
```

## 9. 禁止行为

- 不得先生成答案，再事后拼接证据。
- 不得把第一次搜索失败扩展为现实世界整体不存在。
- 不得把 `没有证据` 转换为 `错误`。
- 不得把一个相似候选项伪装成精确答案。
- 不得把过时信息当作当前事实。
- 当所有候选项都已关闭时，不得生成语言上合理的填充内容。
- 不得声称能够无限搜索。
- 不得隐藏物理或计算限制。
- 不得要求或暴露用户完整的隐藏思维过程。
- 不得用相同确定度表述已验证事实与设计设想。

## 10. 核心伪代码

```text
function negative_first_answer(question, sources):
    frame = normalize(question)
    groups = build_candidate_groups(frame)

    for group in groups:
        candidates = activate(group)

        parallel_for candidate in candidates:
            apply_definition_gate(candidate, frame)
            apply_constraint_gate(candidate, frame)
            apply_time_gate(candidate, frame)
            apply_existence_gate(candidate, sources, frame.scope)
            apply_contradiction_gate(candidate)
            apply_evidence_gate(candidate)
            apply_causality_gate(candidate)
            apply_safety_gate(candidate, frame.risk)

        survivors = candidates where state in {
            ACTIVE_SUPPORTED,
            ACTIVE_TENTATIVE
        }

        if survivors is not empty:
            verified = recheck(survivors)
            if verified is not empty:
                return generate_answer_only_from(verified)

        zero_state = classify_zero(candidates, frame.scope)

        if zero_state is terminal:
            return report_zero_without_fabrication(zero_state)

    return {
        status: UNKNOWN_UNRESOLVED,
        answer: "根据当前可验证范围，尚缺乏足够依据作出判断。"
    }
```

## 11. 技能触发语句

当用户提出以下请求时，激活本技能。

- “核实一下这是否属实。”
- “先确认是否不存在。”
- “检查所有可能原因。”
- “不要产生幻觉地回答。”
- “搜索整份文档。”
- “先排除不满足条件的候选项。”
- “如果不知道，就明确判定为未知。”
- “用这个网状方法回答。”
- “用Negative-First方法分析。”

即使用户没有明确要求，当事实错误风险较高或候选空间很大时，也可以应用本技能。

---

## 12. 完成标准

回答前，应在内部确认：

- 问题范围是否已定义。
- 是否区分了精确候选项与相似候选项。
- 是否优先排除了不成立的候选项。
- 是否区分了证据不足与错误。
- 候选项归零时，是否检查了下一个合理候选组。
- 不存在的断言是否仅限于闭合范围。
- 回答是否只使用存活候选项。
- 剩余不确定性是否对用户可见。
- 是否验证了需要最新性的事实。
- 是否避免用想象填补证据空白。

只有满足所有适用标准后，任务才算完成。
